# Final Reflection

---

## 1. How does Git help in maintaining research traceability?
Git records every change with timestamps and author details, allowing researchers to track the evolution of their work and reproduce previous versions.

---

## 2. Why is GitHub useful for academic collaboration?
GitHub allows multiple researchers to collaborate, review code, manage versions, and track issues efficiently in a centralized platform.

---

## 3. Why should researchers create branches instead of working directly on main?
Branches allow safe experimentation without affecting the stable main branch and ensure only reviewed changes are merged.

---

## 4. What is the role of Pull Requests in collaborative work?
Pull Requests enable discussion, review, and validation of changes before merging into the main branch.

---

## 5. Why are reproducibility files important in research repositories?
They ensure that experiments can be replicated by others, increasing transparency and credibility.

---

## 6. What is the purpose of creating a version tag such as v1.0.0?
It marks a stable version of the project that can be referenced in research and publications.

---

## 7. Why should a repository include a citation file?
It allows others to properly credit the work when used in research.

---

## 8. How does this semester’s Git/GitHub workflow support better PhD research practice?
It enforces structured development, documentation, reproducibility, and collaboration, aligning with research best practices.

---

## Additional Questions

### 1. Why is reproducibility important in PhD-level research?
It ensures results are reliable, verifiable, and can be independently validated.

---

### 2. What is the difference between a commit, a Pull Request, a tag, and a release?
- Commit: snapshot of changes  
- Pull Request: proposal to merge changes  
- Tag: labeled version  
- Release: published version with notes  

---

### 3. Why should code, data, and documentation be version-controlled together?
To maintain consistency and ensure reproducibility of results.

---

### 4. What problems arise if research repositories are poorly documented?
- Difficult to reproduce results  
- Misinterpretation  
- Reduced credibility  
- Poor collaboration  

---

### 5. How can GitHub support thesis work, publications, and collaborative research projects?
It provides version control, collaboration tools, documentation, and reproducibility support.
